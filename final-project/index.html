Tandoori Kitchen — Accessibility Fixes

ATLS 4519 / CSCI 4830 Final Project — Six corrected components from tandoorikitchenco.com

1. Heading Structure

Original Issue: The site uses two <h1> elements ("Home" and "ORDER ONLINE"), places <h4> tags before <h1> in the DOM, and renders all dish names as <h2> with no category grouping — making the heading hierarchy flat and unusable for screen reader navigation.

Fix Applied: A single <h1> for the page, category names as <h2>, and individual dish names as <h3>, creating a clear hierarchy.

2. Image Alt Text

Original Issue: The logo alt text reads "Webp.net-resizeimage" (a file conversion artifact). Food images use meaningless alt text such as "dsaag", "papad", and "Placeholder". The same garlic cheese naan photo is reused for six different naan varieties.

Fix Applied: The logo receives alt="Tandoori Kitchen logo". Each food image gets a unique, descriptive alt attribute that conveys the dish's appearance.

3. Keyboard Focus Indicators

Original Issue: The site only shows focus styles on a few WooCommerce "Add" / "Select" buttons. Navigation links, category tabs, social media icons, the hamburger menu button, and search inputs have no visible focus indicator — making keyboard-only navigation impossible to track.

Fix Applied: A global :focus-visible rule adds a 2 px blue outline with a 2 px offset to every interactive element. Individual components below also receive explicit focus styles.

Press Tab to move through the interactive elements below and verify the blue focus ring.

4. ARIA Button Labels

Original Issue: The site repeats generic "Add" and "Select" buttons 60+ times with no surrounding context. A screen reader user hears "Add, button" repeatedly with no way to distinguish which dish each button corresponds to.

Fix Applied: Each button receives an aria-label="Add [Dish Name] to cart" so screen readers announce the specific item. Visual text remains "Add to Cart" for sighted users.

5. Dietary Abbreviation Legend

Original Issue: The site uses inconsistent notation — "gf", "GF", "gluten free", "Gluten Free", and "GF - DF" appear interchangeably with no legend explaining what the abbreviations mean.

Fix Applied: A visible legend defines each abbreviation. Dish cards use standardized badge-style tags with consistent spelling and aria-label attributes on each abbreviation for screen readers.

GF = Gluten Free

6. Form Labels

Original Issue: The "Find locations near you" modal contains a zip code input with no programmatic label association (missing for/id pairing). The site also renders three redundant "Search for:" inputs, making the page confusing for assistive technology users.

Fix Applied: Every input is paired with a <label> using matching for and id attributes. Redundant search fields are consolidated into a single, clearly labeled input.
